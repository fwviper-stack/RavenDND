Raven Legend

v1.2.0 — archive map updated with Session 002 file class and retrieval order

Document Role: Master archive map for what files exist, what each file is for, and how the archive should decide what to read.
Primary Use Rule: Consult when archive structure, trust level, or document role is unclear.
Archive Model: Mixed-source archive with legacy session material, protocol sessions, and derived support documents under one retrieval framework.

Source Classes

Source Class A — Legacy Session: sessions created before Raven Protocol governed story output. High ambiguity, high contamination risk, preserve explicit facts separately from interpretation.

Source Class B — Protocol Session: sessions played under Raven Protocol, even if the protocol evolves during the session. Read in historical order and do not let newest-version assumptions erase earlier phase meaning.

Source Class C — Derived Support Documents: files built from sessions rather than being sessions themselves. Useful for navigation and state, but not automatically equal to original session truth.

Default Retrieval Guidance

For current state or "what happens next": Character Sheet → Quest Log → Timeline Index → Short Term Memory.

For familiar, apprenticeship, or current household / Edrin dynamics: Timeline Index → Session 002 Extraction Report → Session 002 Transcript Archive only if dialogue precision matters.

For pre-casting mentor conflict or the road to Find Familiar: Timeline Index → Session 001 Extraction Report → Session 001 Transcript Archive.

For formative motive questions: Timeline Index → Long Term Memory → Session 000 bridge set.

For rules / process questions: Raven Protocol before narrative memory files.

Use the smallest sufficient memory depth first; do not default to transcript-level review.

Known Warnings and Gaps

Session 000 remains a legacy source and must not be silently overwritten by later class totals, Chronurgy lock-in, familiar details, or formal spell structure.

Session 001 and Session 002 are both protocol sessions and should be read in historical sequence rather than flattened together into one generic "Find Familiar arc."

Session 002 Transcript Archive is an AI-generated transcript archive, not a literal raw platform export. Use it as fallback and recovery source, not as a perfect raw log surrogate.

The active character sheet currently controls exact current coin, incense, pouch, and familiar state. Memory-layer documents should preserve why those facts matter, not replace the sheet as ledger authority.
